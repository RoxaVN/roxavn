## Testing

RoxaVN sử dụng **Vitest** làm framework testing chính thông qua package `@roxavn/testing`. Skill test-writer hướng dẫn cách viết unit tests và integration tests cho React components và logic trong dự án.

### 1. Nguyên tắc cốt lõi

#### Framework
- Sử dụng **Vitest** cho tất cả các loại test

#### Global APIs
Các hàm sau đã được configured sẵn như globals, **KHÔNG CẦN import**:
- `describe`, `expect`, `it`, `test`
- `beforeEach`, `afterEach`, `beforeAll`, `afterAll`

#### Mocking
- Sử dụng **`vi`** (utility có sẵn của Vitest) cho tất cả nhu cầu mocking và spying

#### File Naming
- Đặt tên file test: `*.test.ts` hoặc `*.test.tsx`
- Đặt file test **cạnh file source** tương ứng

#### ESM
- Sử dụng extension `.js` trong các import local file

---

### 2. Các loại test

| Loại | Tool sử dụng | Mục đích |
|------|-------------|----------|
| **Server Services** | `setUpDatabase` từ `@roxavn/core/server/testing` | Test logic phía server, database operations |
| **Web Pages** | `PageTester` từ `@roxavn/core/web/testing` | Test UI/React pages với flows chuẩn |
| **Pure Logic** | Vitest standard features | Test shared code, utilities, base logic |

### 3. Testing Server Services

#### Pattern cơ bản

```typescript
import { setUpDatabase } from '@roxavn/core/server/testing';
import {
  CreateUserService,
  GetUserService,
  UpdateUserService,
} from './user.js';

describe('user services', async () => {
  await setUpDatabase();

  it('should create and get user', async () => {
    // Mỗi service được lấy instance riêng qua .getInstance()
    const createService = await CreateUserService.getInstance();
    const userResult = await createService.handle({
      username: 'testuser',
      displayName: 'Test User',
    });
    expect(userResult.id).toBeDefined();

    // Dùng service khác để get user vừa tạo
    const getService = await GetUserService.getInstance();
    const fetchedUser = await getService.handle({ id: userResult.id });
    expect(fetchedUser.username).toBe('testuser');
  });

  it('should update user', async () => {
    const createService = await CreateUserService.getInstance();
    const user = await createService.handle({
      username: 'updateuser',
      displayName: 'Update User',
    });

    const updateService = await UpdateUserService.getInstance();
    await updateService.handle({
      id: user.id,
      displayName: 'Updated Name',
    });

    const getService = await GetUserService.getInstance();
    const updatedUser = await getService.handle({ id: user.id });
    expect(updatedUser.displayName).toBe('Updated Name');
  });
});
```

#### Database Setup

- `setUpDatabase` khởi tạo **temporary PostgreSQL container** và chạy tất cả migrations
- Database được cleanup tự động sau khi test xong (trong `afterAll`)
- Mỗi test suite sẽ có database sạch, independent

#### Singleton Pattern

- Services được access qua **`ServiceName.getInstance()`**
- Đảm bảo single instance xuyên suốt test
- Mỗi lần gọi `getInstance()` trả về cùng một instance

#### Cấu trúc Service trong Roxavn

Mỗi service thường:
- Được decorate bằng `@useApi()` hoặc `@injectable()`
- Mở rộng `ResourceService` hoặc `InjectDatabaseService`
- Có method `handle()` để thực hiện logic

```typescript
// Ví dụ cấu trúc một service (từ module-user)
@useApi(userApi.create)
export class CreateUserService extends ResourceService {
  async handle(request: InferApiRequest<typeof userApi.create>) {
    return super.create('user', request);
  }
}

@useApi(userApi.getOne)
export class GetUserService extends ResourceService {
  async handle(request: InferApiRequest<typeof userApi.getOne>) {
    return super.getOne('user', request);
  }
}

@useApi(userApi.update)
export class UpdateUserService extends ResourceService {
  async handle(request: InferApiRequest<typeof userApi.update>) {
    return super.update('user', request);
  }
}
```

### 4. Testing Web Pages

#### 4.1. Page Testing (Standard CRUD)

Với các admin pages chuẩn, sử dụng `PageTester` để tự động test các flows.

```typescript
import { PageTester } from '@roxavn/core/web/testing';
import { myPage } from './myPage.js';

describe('myPage', () => {
  it('should support standard flows', async () => {
    const pageTester = new PageTester(myPage);
    
    await pageTester.testFilter();
    await pageTester.testAdd();
    await pageTester.testItemActions([0, 1]); // test actions cho 2 items đầu
  });
});
```

#### PageTester Methods

| Method | Mô tả |
|--------|-------|
| `testFilter()` | Test chức năng filter/search |
| `testAdd()` | Test flow thêm item mới |
| `testItemActions([indices])` | Test actions cho các items được chỉ định |
| `testPagination()` | Test phân trang |
| `testEdit()` | Test flow chỉnh sửa item |

#### 4.2. Component Testing (Custom UI)

Với custom components không theo chuẩn CRUD, sử dụng `render` từ `@roxavn/core/web/testing`.

```typescript
import { render } from '@roxavn/core/web/testing';
import { MyComponent } from './MyComponent.js';

describe('MyComponent', () => {
  it('handles interaction', async () => {
    const screen = await render(<MyComponent />);
    
    const button = screen.getByRole('button');
    await button.click();
    
    expect(button).toBeDisabled();
  });
});
```

#### Render Features

- Tự động wrap component với **Mantine** và **API providers**
- Cung cấp `screen` object để query elements
- Hỗ trợ async state changes với `vi.waitFor`

#### 4.3. Ví dụ thực tế - Web Component

```typescript
import { render, screen } from '@roxavn/core/web/testing';
import { UserCard } from './UserCard.js';
import { vi } from 'vitest';

describe('UserCard', () => {
  it('should display user information', async () => {
    await render(<UserCard user={{ name: 'John Doe', email: 'john@example.com' }} />);
    expect(screen.getByText('John Doe')).toBeInTheDocument();
  });

  it('should call onEdit when button clicked', async () => {
    const onEdit = vi.fn();
    await render(<UserCard user={{ name: 'Jane' }} onEdit={onEdit} />);
    await screen.getByRole('button', { name: /edit/i }).click();
    expect(onEdit).toHaveBeenCalledTimes(1);
  });
});
```

#### 4.4. Service/API Mocking

##### Sử dụng `ApiMocker`

```typescript
import { ApiMocker } from '@roxavn/core/web/testing';

const apiMocker = new ApiMocker();
apiMocker.mock(api, ({ response }) => ({
  ...response,
  items: [{ id: '1', name: 'Test' }],
}));
```

##### Sử dụng `vi.mock`

```typescript
vi.mock('../module.js', () => ({
  fn: vi.fn().mockResolvedValue('value'),
}));

// Hoặc mock service cụ thể
vi.mock('../services/user.service.js', () => ({
  UserService: {
    getInstance: vi.fn().mockResolvedValue({
      getUsers: vi.fn().mockResolvedValue([
        { id: '1', name: 'Mocked User' },
      ]),
    }),
  },
}));
```

### 5. Testing Pure Logic

#### Pattern cơ bản

Với shared code, base logic, và utilities, chỉ cần dùng standard Vitest features - không cần database hay DOM rendering.

```typescript
import { myUtility } from './utils.js';

describe('myUtility', () => {
  it('handles transformations correctly', () => {
    const result = myUtility('input');
    expect(result).toBe('output');
  });

  it('handles async logic', async () => {
    const result = await myUtility('async-input');
    expect(result).toBe('async-output');
  });
});
```

#### Ví dụ thực tế - Pure Logic

```typescript
// utils/formatCurrency.test.ts
import { formatCurrency, parseCurrency } from './formatCurrency.js';

describe('formatCurrency', () => {
  it('should format USD correctly', () => {
    expect(formatCurrency(1000)).toBe('$1,000.00');
  });

  it('should format EUR correctly', () => {
    expect(formatCurrency(1000, 'EUR')).toBe('€1,000.00');
  });
});

describe('parseCurrency', () => {
  it('should parse currency string', () => {
    expect(parseCurrency('$1,234.56')).toBe(1234.56);
  });
});
```

### 6. Mocking

#### 6.1. Mocking với `vi.mock`

```typescript
// Mock một module hoàn toàn
vi.mock('../module.js', () => ({
  fn: vi.fn().mockResolvedValue('value'),
  default: vi.fn().mockReturnValue({}),
}));

// Mock module với implementation tùy chỉnh
vi.mock('../user.service.js', () => ({
  UserService: {
    getInstance: vi.fn().mockResolvedValue({
      getUser: vi.fn().mockResolvedValue({ id: '1', name: 'Test' }),
      createUser: vi.fn().mockImplementation((data) => Promise.resolve({ id: '2', ...data })),
    }),
  },
}));
```

#### 6.2. Mock Functions

```typescript
// Mock return value
const mockFn = vi.fn().mockReturnValue('result');

// Mock resolved value (async)
const asyncMockFn = vi.fn().mockResolvedValue({ data: 'value' });

// Mock rejected value
const errorMockFn = vi.fn().mockRejectedValue(new Error('Failed'));

// Mock implementation
const implFn = vi.fn().mockImplementation((arg) => arg * 2);

// Reset mock
beforeEach(() => {
  vi.clearAllMocks();
});
```

#### 6.3. Async UI Changes

```typescript
// Đợi UI update sau async operations
await vi.waitFor(() => expect(el).toBeVisible());

// Đợi với timeout
await vi.waitFor(
  () => expect(screen.queryByText('Loading...')).not.toBeInTheDocument(),
  { timeout: 2000 }
);
```

#### 6.4. Timer Mocking

```typescript
// Fake timers cho setTimeout/setInterval
vi.useFakeTimers();

it('should debounce input', () => {
  const callback = vi.fn();
  const debouncedFn = debounce(callback, 300);
  
  debouncedFn();
  debouncedFn();
  debouncedFn();
  
  vi.advanceTimersByTime(300);
  
  expect(callback).toHaveBeenCalledTimes(1);
});

vi.useRealTimers(); // Khôi phục real timers
```

### 7. Chạy Tests

#### Chạy một file cụ thể

```bash
npx roxavn-test path/to/file.test.ts
```



### 8. Tổng kết

#### Quick Reference

| Loại Test | Setup | Imports | Công cụ |
|-----------|-------|---------|---------|
| **Server Service** | `await setUpDatabase()` | `@roxavn/core/server/testing` | Service.getInstance() |
| **Web Page (CRUD)** | `new PageTester(page)` | `@roxavn/core/web/testing` | PageTester methods |
| **Web Component** | `await render(<Component />)` | `@roxavn/core/web/testing` | screen, fireEvent, vi.waitFor |
| **Pure Logic** | None needed | vitest globals | expect, vi.fn(), vi.mock() |

#### Best Practices

1. **Mỗi test nên độc lập** - Không phụ thuộc vào thứ tự chạy
2. **Sử dụng descriptive names** - Tên test mô tả rõ what/when/then
3. **Test behavior, không implementation** - Tập trung vào kết quả
4. **Mock external dependencies** - Không gọi API thực hoặc database thật
5. **Clean up sau mỗi test** - Reset mocks, clear state
6. **Đặt file test cạnh source** - Dễ tìm, dễ maintain
