## Stories - Hệ thống tài liệu RoxaVN

RoxaVN sử dụng **Story Files** để tự động sinh tài liệu cho các **components**, **utilities**, **hooks** và **services**. Mỗi Story File export một **story object** dùng để định nghĩa nội dung tài liệu và cấu hình phần xem trước (preview).

Trong quá trình build tài liệu, framework sẽ quét mã nguồn để trích xuất thông tin như API, kiểu dữ liệu, JSDoc và các metadata liên quan, sau đó tự động inject các dữ liệu này vào Story File. Vì vậy, Story File phải tuân thủ quy ước đặt tên tương ứng với file nguồn. Ví dụ, nếu file nguồn là `Button.tsx` thì Story File phải được đặt tên là `Button.story.tsx`. Một điểm khác biệt là framework có thể tự động trích xuất mã nguồn từ thẻ `@example` trong JSDoc để tạo playground tương tác trong tài liệu. Nhờ đó, ví dụ sử dụng được định nghĩa ngay tại mã nguồn và luôn đồng bộ với tài liệu được sinh ra.

Ngoài Story File dạng TypeScript, RoxaVN cũng hỗ trợ Story File dạng Markdown với phần mở rộng `.story.mdx`. Các file này chỉ chứa nội dung tài liệu tĩnh và không liên kết với mã nguồn cụ thể, do đó không tham gia vào quá trình trích xuất hoặc inject metadata từ source code.

### 1. Các loại Story

#### 1.1. ModuleStory - Document Utilities/Functions

Dùng để document các functions, classes, constants được export từ **base** module.

**Cú pháp:**

```typescript
import { ModuleStory } from '@roxavn/stories/web';
import { myFunction, myConstant, MyClass } from './module.js';

export default ModuleStory({
  exports: {
    myFunction,
    myConstant,
    MyClass,
  },
  playgroundProps: {
    scopes: { /* dependencies */ },
  },
});
```

**Cách viết tài liệu**

```typescript
/**
 * Mô tả chức năng của function
 * 
 * @example
 * return myFunction();
 */
export function myFunction() {
  // Implementation
}
```

Tài liệu nên gồm 2 phần:
- **Description**: Mô tả chức năng của function
- **@example**: Ví dụ minh họa cách sử dụng một function. Để kết quả được hiển thị trong playground, đoạn mã ví dụ bắt buộc phải có câu lệnh **return**. `ModuleStory` sẽ tự động tạo một playground tương tác từ đoạn mã ví dụ, cho phép bạn chỉnh sửa và thực thi mã trực tiếp để kiểm tra các trường hợp sử dụng khác nhau. Nếu ví dụ sử dụng các dependency bên ngoài, chúng cần được khai báo trong thuộc tính `playgroundProps.scopes` của story để playground có thể truy cập và thực thi chính xác các giá trị đó.


Ví dụ với dependencies

```typescript
// src/web/base/my.class.ts
export class MyClass {
  /**
   * Mô tả cho method
   * 
   * @example
   * return new MyClass().method(user.name);
   */
  public method(name: string) {
    return `Hello, ${name}`;
  }
}
```

```typescript
// src/web/base/my.class.story.tsx
import { ModuleStory } from '@roxavn/stories/web';
import { MyClass } from './my.class.js';

export default ModuleStory({
  exports: { MyClass },
  playgroundProps: {
    scopes: { user: { name: 'John' } },
  },
});
```

#### 1.2. ComponentStory - Document React Components

Dùng để tài liệu hóa các React component kèm theo playground tương tác, cho phép người dùng chỉnh sửa mã nguồn và xem kết quả hiển thị theo thời gian thực.

Playground được xây dựng dựa trên thư viện React Live, vì vậy hầu hết các quy tắc và khả năng của React Live đều được hỗ trợ, bao gồm việc truyền dependencies thông qua `scopes`, sử dụng JSX trực tiếp và tùy chọn thực thi mã. React Live cung cấp khả năng chỉnh sửa mã và xem trước kết quả ngay lập tức trong cùng một giao diện.

Để hiểu rõ hơn về các tính năng và cách sử dụng playground, tham khảo tài liệu chính thức của [React Live](https://nearform.com/open-source/react-live/docs/)


**Cú pháp:**

```typescript
import { ComponentStory } from '@roxavn/stories/web';
import { MyComponent } from './MyComponent.js';

export default ComponentStory({
  component: MyComponent,
  playgroundProps: {
    scopes: { /* dependencies như Mantine components, APIs */ },
    noInline: true, // nếu set là true thì component phải gọi render() để hiển thị
  },
});
```

**Ví dụ**

```ts
// src/web/components/ActionButton/ActionButton.tsx

/**
 * A button that performs an asynchronous action when clicked. It shows a loading state while the action is in progress, and handles errors by showing an error modal. The `onClick` prop should be a function that returns a promise.
 *
 * @example
 * <ActionButton onClick={async () => {
 *   await new Promise(resolve => setTimeout(resolve, 2000));
 *   alert('Action completed!');
 * }}>
 *   Click me
 * </ActionButton>
 */
export function ActionButton() {
  // implement here ...
}
```

```tsx
// src/web/components/ActionButton/ActionButton.story.tsx
import { ComponentStory } from '@roxavn/stories/web';
import { ActionButton } from './ActionButton.js';

export default ComponentStory({ component: ActionButton });
```

#### 1.3. HookStory - Document React Hooks

Dùng để tài liệu hóa các custom React hooks. HookStory hoạt động tương tự ComponentStory, cho phép hiển thị tài liệu API, ví dụ sử dụng và playground tương tác để thử nghiệm hook trực tiếp.

**Ví dụ:**

```typescript
// src/web/hooks/useAuthUser.tsx

/**
 * Returns the current authenticated user.
 *
 * @example
 * () => {
 *   const user = useAuthUser();
 *   return <div>{user ? `Hello ${user.username}` : 'Not logged in'}</div>;
 * }
 */
export function useAuthUser() {
  // implement here ...
}
```

```typescript
// src/web/hooks/useAuthUser.story.tsx
import { HookStory } from '@roxavn/stories/web';
import { useAuthUser } from './useAuthUser.js';

export default HookStory({
  hook: useAuthUser,
});
```

#### 1.4. ServerStory - Document Server Services

Dùng để tài liệu hóa các server-side service kèm theo khả năng thực thi và kiểm thử trực tiếp từ giao diện tài liệu.

Framework sẽ tự động trích xuất mã nguồn từ thẻ @example trong phương thức handle() của service để tạo playground tương tác. Trong môi trường playground, `service` đã được inject sẵn, do đó bạn có thể gọi trực tiếp các phương thức của service.

Khi thực thi ví dụ từ trình duyệt, playground sẽ gửi một request đến server để yêu cầu service tương ứng thực thi đoạn mã ví dụ. Kết quả sau đó được trả về cho client và hiển thị trực tiếp trong giao diện tài liệu. Điều này cho phép kiểm tra nhanh hành vi thực tế của service.

Do các ví dụ được thực thi trên server và có thể tương tác trực tiếp với cơ sở dữ liệu hoặc các tài nguyên hệ thống khác, tính năng này chỉ nên được sử dụng với database ở môi trường development hoặc các môi trường kiểm thử được kiểm soát.


**Ví dụ**

```typescript
// src/server/services/user.ts
export class GetUsersService {
  /**
   * Get all users
   * 
   * @example
   * return service.handle({type: 'bot'});
   */
  handle(request: any) {
    // implement here ...
  }
}
```

```typescript
// src/server/services/user.story.tsx
import { createAction } from '@roxavn/stories/server';
import { ServerStory } from '@roxavn/stories/web';
import * as services from './user.js';

export default ServerStory({});

export const action = createAction(services);
```

#### 1.5. MDX Stories - Tài liệu tĩnh

Dùng cho tài liệu chi tiết với nội dung tùy chỉnh.

**Cú pháp:**

```markdown
# ComponentName

Description of the component...

```tsx
import { Component } from '@roxavn/core/web';

<Component prop="value" />
```
```

**Ví dụ:**

```mdx
// src/web/components/layout/VirtualList.story.mdx
# VirtualList

RoxaVN re-export of virtua components. You can use it to create virtualized lists and grids.

Check [virtua documentation](https://github.com/inokawa/virtua) for more information.

```tsx
import { ScrollableDiv, Virtualizer } from '@roxavn/core/web';

<ScrollableDiv>
  <Virtualizer>
    {conversations.map((item) => (
      <NavLink
        variant="light"
        active={item.id === conversation?.id}
        label={<div>{item.name}</div>}
      />
    ))}
  </Virtualizer>
</ScrollableDiv>
```

### 2. Cấu trúc Story Files

#### 2.1. Vị trí đặt file

Story files được đặt cạnh source files:

```
src/
├── base/
│   ├── utils/
│   │   ├── string.ts
│   │   └── string.story.tsx    ← story cho string utilities
│   ├── errors.ts
│   └── errors.story.tsx        ← story cho errors
│
├── server/
│   └── services/
│       ├── user.ts
│       └── user.story.tsx      ← story cho user services
│
└── web/
    ├── components/
    │   ├── api/
    │   │   ├── ApiTable.tsx
    │   │   └── ApiTable.story.tsxtsx
    │   └── layout/
    │       ├── DrawerTrigger.tsx
    │       ├── DrawerTrigger.story.tsx
    │       └── VirtualList.story.mdx  ← MDX story
    │
    └── hooks/
        ├── useAuthUser.ts
        └── useAuthUser.story.tsx
```

#### 2.2. Quy tắc đặt tên

- File story: `<name>.story.tsx` hoặc `<name>.story.mdx`
- Export default: Story object phù hợp với loại item được document
- Server stories: Export thêm `action` từ `createAction`

### 3. Cách sử dụng

Để khởi động Story Server trong môi trường development, chạy lệnh:

```bash
npx roxavn stories
```

Lệnh này sẽ khởi tạo máy chủ tài liệu và tự động theo dõi thay đổi của mã nguồn để cập nhật nội dung tài liệu trong quá trình phát triển.

Để build tài liệu cho môi trường production, chạy:

```bash
npx roxavn stories:build
```

Lệnh trên sẽ sinh ra phiên bản tĩnh của tài liệu cùng các tài nguyên cần thiết để triển khai. Sau khi quá trình build hoàn tất, có thể khởi động máy chủ để phục vụ phiên bản production bằng lệnh:

```bash
npx roxavn stories:serve
```

### 4. Tổng kết

| Story Type | Use for | Default Export |
|------------|---------|----------------|
| **ModuleStory** | Functions, classes, constants, utilities | `ModuleStory({ exports: {...} })` |
| **ComponentStory** | React components | `ComponentStory({ component: Component })` |
| **HookStory** | React hooks | `HookStory({ hook: useHook })` |
| **ServerStory** | Server services | `ServerStory({})` + `export const action = createAction(services)` |
| **MDX** | Detailed docs with custom content | Markdown file |

### 5. Checklist khi viết Story

- [ ] Đặt file story cạnh source file tương ứng
- [ ] Sử dụng đúng loại story cho item
- [ ] Import item cần document
- [ ] Export default story object
- [ ] Thêm `playgroundProps.scopes` nếu cần dependencies
- [ ] Với server stories: export `action` từ `createAction`
- [ ] Với MDX stories: viết documentation chi tiết với code examples
